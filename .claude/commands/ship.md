작업 완료 체크리스트를 순서대로 실행한다.

## 실행 순서

### 1단계 — 코드 검토 (bug-fixer 에이전트)

최근 변경된 코드를 대상으로 bug-fixer 에이전트를 실행한다.
- `git diff HEAD` 또는 `git status`로 변경 파일 목록을 파악한다.
- 변경된 파일들에 대해 bug-fixer 에이전트를 호출하여 버그를 검토하고 수정한다.
- 버그가 발견되어 수정된 경우, 수정 내용을 사용자에게 보고한다.

### 2단계 — 문서 동기화 (docs-manager 에이전트)

docs-manager 에이전트를 실행한다.
- 변경된 기능에 해당하는 `DOCS.md`, `docs/guide/`, `docs/api/` 를 업데이트한다.
- `docs/.vitepress/config.ts` 사이드바도 필요 시 수정한다.
- 최종 업데이트 날짜를 오늘로 갱신한다.

### 3단계 — 커밋 및 푸시

모든 변경 파일(소스 + 문서)을 함께 커밋하고 푸시한다.

```bash
git add <변경된 파일들>
git commit -m "..."
git push
```

커밋 메시지는 변경 내용을 요약하여 작성한다 (feat/fix/docs/refactor 등 prefix 사용).

## 완료 보고

모든 단계가 끝나면 아래 형식으로 보고한다:

```
## Ship 완료

- [x] 코드 검토 완료 (버그 N개 수정 / 이상 없음)
- [x] 문서 동기화 완료
- [x] 커밋 및 푸시 완료 (커밋 해시: xxxxxxx)
```
